1. heroku login -i
2. heroku create
3. do locally any changes
4. app.js: mongoose.connect(<cluster url>)
				cluster url: mongodb.com/cloud/atlas => clusters => connect => copy url
		   in cluster url replace <password> with dbUser's password (only letters/numbers!)
		   in mongoose.connect object add: useCreateIndex: true
		   + const port = process.env.PORT || 3000 		//(for the app.listen)
5. git init, etc..
6. git push heroku master
7. run app: https://<app_name>.herokuapp.com  	(heroku => dashboard => app => open app)
8. heroku logs		// to detect errors

9. It's better to have seperate dbs locally and on heroku, so:
	on terminal we create a local environment variable for the db url:
		export DATABASEURL=mongodb://localhost/yelp_camp
	in app.js:
		mongoose.connect(process.env.DATABSEURL);
	git add /commit /push heroku master
	set a heroku environment variable for the db url with the same name as the local one:
		a) heroku => app => settings => reveal config vars => add KEY & VALUE (name & url) => add
		b) on terminal:		heroku config:set DATABASEURL=<url>
   (+) DATABASEURL is hidden => even by adding code to github, our url will still be private
